01 — WHAT TO SHARE

Three things, that's it.

Frontend

A staging URL (or a build we can host) the agents can drive. Optional repo for deeper context, and a test account to log in.

Backend

Your API base + OpenAPI/GraphQL spec — or a container / docker-compose we spin up in the sandbox. Source repo optional, for tracing.

Data

A schema plus an anonymized or synthetic seed so flows have realistic state. We require no real PII.

02 — THE TEMPLATE

One file describes it all.

Drop a matraix.yaml in your repo (or send it over) — it points us at your frontend, backend, and data, and tells the agents what to evaluate.

matraix.yaml
# matraix.yaml — describe your stack for the agents
project: acme-checkout

frontend:
  url: "https://staging.acme.com"      # live URL the agents drive
  repo: "github.com/acme/web"          # optional, for context
  auth: { type: test-account, user: "bot@acme.com" }

backend:
  api:     "https://staging.acme.com/api"
  spec:    "openapi.yaml"              # OpenAPI / GraphQL schema
  sandbox: "docker-compose.yml"        # or a container image to run
  repo:    "github.com/acme/api"       # optional source for tracing

data:
  schema: "schema.sql"               # table / model definitions
  seed:   "anonymized_sample.csv"     # synthetic or anonymized rows
  pii:    none                        # we require no real PII

goals:
  - flow: checkout
    success: order_confirmed
    metrics: [conversion rate, latency, error rate]

segments: [mobile, new_users, non-native English]

03 — HOW IT WORKS

From your repo to results.

01Share

Send your matraix.yaml — frontend URL, backend spec, and an anonymized data seed. That's all we need to start.

02Sandbox

We stand up an isolated, per-customer sandbox of your stack — network-restricted, no production access, torn down after the run.

03Simulate

Millions of persona-weighted agents run your real flows end-to-end across 1,162 dimensions — the long tail included.

04Report

You get a segmented report — friction, lift, and failures by segment — plus full replays and training-ready trajectories.

04 — YOUR CODE & DATA

Where it runs, and who can see it.

Q01Where does our code run?+
In an isolated, per-customer sandbox — network-restricted and never co-mingled with other customers. It's torn down after the run, and you can request deletion at any time.
Q02Do you need real production data?+
No. A schema plus an anonymized or synthetic seed is enough to give flows realistic state. We require no real PII, and any inputs you share are handled under NDA.
Q03Who can access it, and can we revoke it?+
Scoped test credentials only, encrypted in transit and at rest, accessible to your assigned team under NDA. Access is revocable and deletable on request, and your code is never used to train shared models without explicit opt-in.
Q04What if we can't share source code?+
You don't have to. A staging URL + API spec is enough for the agents to drive your product as a black box. Source repos are optional — they only let us trace failures deeper.
